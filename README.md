# Registration Form with Client-Side Validation

A responsive user registration form built using **HTML**, **CSS**, and **JavaScript**. The project demonstrates form design, custom styling, input validation, and real-time user feedback without requiring a backend server.

## Features

* User registration form with:
  * Username field
  * Email field
  * Password field
  * Confirm Password field
  * Gender selection using custom-styled radio buttons
* Client-side form validation:
  * Email format verification
  * Password length validation
  * Password confirmation matching
* Error messages displayed beneath invalid fields
* Visual feedback using color-coded input borders
* Responsive layout for desktop and mobile devices
* Custom CSS-styled radio buttons

## Technologies Used

* **HTML5** – Form structure and content
* **CSS3** – Styling, layout, responsive design, and custom form controls
* **JavaScript** – Input validation and user interaction

## Validation Rules

The form validates the following conditions before submission:

* Username cannot be empty
* Email cannot be empty and must follow a valid email format
* Password cannot be empty and must contain at least 8 characters
* Confirm Password cannot be empty and must match the Password field

If any validation fails, the form submission is prevented and an appropriate error message is displayed.
